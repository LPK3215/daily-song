# 📝 配置文件使用说明

## 🎵 两个配置文件

### 1. `date-songs.json` - 按日期指定歌曲（优先级最高）

指定某个日期播放某首歌，适合节日、纪念日等特殊场合。

**格式**：
```json
{
  "YYYY-MM-DD": ["歌名", "歌手", "音频文件", "封面(可选)", "寄语(可选)"]
}
```

**示例**：
```json
{
  "2026-06-13": ["空白", "佚名", "空白.mp3"],
  "2026-06-14": ["晴天", "周杰伦", "晴天.mp3", "晴天.jpg"],
  "2026-06-15": ["七里香", "周杰伦", "七里香.mp3", "七里香.jpg", "经典老歌"]
}
```

**字段说明**：
- **位置1 - 歌名**（必填）：歌曲标题
- **位置2 - 歌手**（必填）：歌手名字
- **位置3 - 音频文件**（必填）：
  - 本地文件：`歌曲.mp3`（自动补全 `media/audio/` 前缀）
  - 外部链接：`https://example.com/song.mp3`
- **位置4 - 封面**（可选）：
  - 本地文件：`封面.jpg`（自动补全 `media/covers/` 前缀）
  - 外部链接：`https://example.com/cover.jpg`
  - 留空：`""`
- **位置5 - 寄语**（可选）：显示在页面底部的文字

---

### 2. `default-songs.json` - 轮播列表（备用）

当 `date-songs.json` 中没有指定今天的歌曲时，从这里轮播。

**格式**：
```json
{
  "anchor": "YYYY-MM-DD",
  "songs": [
    ["歌名", "歌手", "音频文件", "封面(可选)", "寄语(可选)"],
    ...
  ]
}
```

**示例**：
```json
{
  "anchor": "2026-06-13",
  "songs": [
    ["空白", "佚名", "空白.mp3"],
    ["晴天", "周杰伦", "晴天.mp3", "晴天.jpg"],
    ["七里香", "周杰伦", "七里香.mp3", "七里香.jpg", "经典老歌"]
  ]
}
```

**轮播规则**：
- `anchor`（锚点日期）：从这一天开始轮播
- 第0天 = songs[0]，第1天 = songs[1]...
- 播完后循环

## 📋 快速上手

### 添加一首歌（指定日期）

编辑 `date-songs.json`：
```json
{
  "2026-12-25": ["Last Christmas", "Wham!", "last-christmas.mp3", "xmas.jpg", "圣诞快乐！"]
}
```

### 添加一首歌（轮播列表）

编辑 `default-songs.json`：
```json
{
  "anchor": "2026-06-13",
  "songs": [
    ["空白", "佚名", "空白.mp3"],
    ["新歌", "新歌手", "new-song.mp3"]  ← 添加这一行
  ]
}
```

---

## ⚠️ 注意事项

1. **JSON 语法**：
   - 最后一项后面不能有逗号
   - 所有字符串用双引号 `""`
   - 数组用方括号 `[]`

2. **文件路径**：
   - 音频文件放在 `media/audio/` 目录
   - 封面图片放在 `media/covers/` 目录
   - 只需写文件名，路径会自动补全

3. **外部链接**：
   - 支持 YouTube、Bilibili、Spotify 等（自动识别为 embed）
   - 支持直接 MP3 链接（自动识别为 url）

## 🔄 旧格式兼容

旧的对象格式仍然支持：
```json
{
  "2026-06-13": {
    "title": "空白",
    "artist": "佚名",
    "source": "local",
    "src": "media/audio/空白.mp3",
    "cover": "",
    "note": ""
  }
}
```

建议迁移到新格式，更简洁易读！
