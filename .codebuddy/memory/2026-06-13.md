## 架构决策讨论
- 数据存储方案确定：使用单一 `schedule.json` 文件，不拆年月，不用数据库（SQLite/MySQL/CloudBase等）
- 理由：纯静态 GitHub Pages 部署 + 小数据量（年55KB/十年550KB）+ 零外部依赖 + git 天然归档
- 未来管理页面/定期播放/历史记录均读写同一个 schedule.json，系统层只读、操作层只写
- 浏览器无法直接写服务端文件，未来写入需通过 git commit/push 或接入 BaaS API
